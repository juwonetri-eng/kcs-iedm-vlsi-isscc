### 2단계: 논문 내용 검토
각 논문의 제목과 내용을 읽고:
1. **물리층 기술** (소자, 공정, 소재) → 소자·공정
2. **기능층 기술** (회로, 칩, 제어) → 회로·시스템
3. **신 패러다임** (양자, 신경형) → 교차·신흥

### 3단계: 현재 분류와 비교
- 일치 → 분류 정확
- 불일치 → 재분류 필요

## 재분류된 논문

### 분과 K: 메모리 (2편)

#### 1. Processing-In-Memory Based Cross-Point Arrays
- **원래 분류**: 소자·공정
- **재분류**: **회로·시스템** ✓
- **근거**: 메모리 내 연산 알고리즘 (Tiki-Taka) / 메모리 시스템 최적화
- **분석**: 소자(cross-point array) 자체가 아니라, **메모리 제어/알고리즘 설계**

#### 2. A Configurable Processing-in-Memory Evaluation Framework
- **원래 분류**: 소자·공정
- **재분류**: **회로·시스템** ✓
- **근거**: PIM 시스템 평가 프레임워크 / 아키텍처 최적화
- **분석**: 메모리 내 연산 **시스템/회로** 설계

## 분류 원칙 검증

### 규칙 기반 분류의 정확성

**현재 방식: 분과(division) 기반 일괄 할당**

```
KCS 분과 (22개) → KCS_PRIOR (도메인 매핑) → 3개 Tier1 도메인
```

**장점**:
- ✓ 객관적, 일관성 있음
- ✓ 94% 자동화 가능 (키워드 불필요)
- ✓ KCS 공식 분과 구조 반영

**한계**:
- ✗ 개별 논문 세부 성격 미반영 (~1%)
- ✗ 경계 사례 (메모리 소자 vs 메모리 제어기) 모호

### 권장사항

현재 분류 체계를 **유지**하되, **다음 조정**:

1. **분과 K (메모리)**
   - PIM/메모리 시스템 관련 논문 → 회로·시스템
   - 신소자(ReRAM, MRAM 소자) → 소자·공정 유지
   - 경계 기준: "제어/알고리즘" vs "소자 제조"

2. **분과 H (디스플레이)**
   - 디스플레이 소자 제조 → 소자·공정 유지
   - 센서/이미징 회로 설계 → 회로·시스템 (소수)

3. **분과 G (모델링)**
   - 기존 분류 유지 (모델링은 다학제적)
